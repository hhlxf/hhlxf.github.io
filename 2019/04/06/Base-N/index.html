Base N算法的实现

最近在题目中遇到了魔改的base家族，但是，原以为已经了解base家族的我竟然都没有认出来是base家族的算法，实在是菜的真实，因此再花一些时间实现一下base家族的算法

Base64

base64将每三个字节编码为4个字符，每6个bit决定一个字符，因此需要将输入的长度补全为3的倍数，下面这张图能很好的解释

逆向识别

  • 识别base家族的关键还是在于table表的识别，其次就是对这个算法的熟悉程度，比如说补齐成指定长度的操作，还有就是长度的收缩(解码)和扩展(编码)
  • 解码的识别相对要麻烦一些，有很多实现方式，例如在解码过程中可以预先把每个字符在表中的位置存下来，就不需要调用index函数，也可以像编码那样，采用移位与或运算截取其中的片段，而不需要组合成24bit
